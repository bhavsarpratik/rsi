Frontier agents post-train a model autonomously in 10 hours on one GPU. Best agent hits 23.2% vs 51.1% for human experts, and some cheat with test-set training and stray API keys.

Paper 05 Qu & Lu · arXiv:2603.23420 · 2026

Bilevel Autoresearch: Meta-Autoresearching Itself

An outer loop reads the inner autoresearch loop's code and writes new search mechanisms as Python at runtime. Mechanism research gives a 5x gain over the inner loop alone.

Autoresearch Bilevel Code generation

Harness evolution Observability Falsifiable edits

Freezes the model and evolves a lifecycle-aware harness. Improves 116 of 126 settings at 88.5% average relative gain, and a harness evolved on a 4B model transfers to 17 others.

Harness adaptation Frozen model Transfer

Trains a skill document like a parameter: bounded edits, a learning-rate budget, a held-out gate. Best or tied on all 52 cells, +24.8 points on GPT-5.5 inside Codex.

Agent skills Text-space optimizer Transfer
Paper 09 Hexo Labs · arXiv:2605.27276 · 2026

SIA: Self Improving AI with Harness & Weight Updates

A Feedback-Agent updates both the harness and the model weights in one loop. Beats scaffold-only iteration on three unrelated tasks and clears prior SOTA on all three.

Models memory as a heterogeneous graph and evolves its topology, repairing links, pruning interference, and distilling recurring successes into reusable circuits. SOTA on three benchmarks.

Paper 11 Harbin Institute of Technology · arXiv:2605.29682 · 2026

Counts only feedback that is informative, valid, non-redundant, and retained. This coordinate predicts success far better than raw compute (R2 up to 0.99 vs 0.33).

Paper 15 Darwin Agent Team · arXiv:2606.14249 · 2026

HarnessX: A Composable, Adaptive, and Evolvable Agent Harness Foundry

Treats the harness as a typed object: composes it from processors, evolves it from traces under RL-inspired guardrails (AEGIS), and co-evolves the model. +14.5% average across five benchmarks.

Composable harness AEGIS Co-evolution

How to read these

Every note follows the same shape: a one-paragraph summary, the problem the paper attacks, a flow diagram of the architecture, the results in a table, what each design choice changes, where it sits among prior work, the limitations, and a learnings plus pros-and-cons close. The goal is to make the papers comparable to each other, not just readable on their own.
